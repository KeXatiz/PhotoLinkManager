- Class vs Object (for reference https://www.w3schools.com/java/java_classes.asp)
class(smtg like template/blueprint) has variable/attributes and it can do methods/functions
object(smtg like real thing built from that class) - an object actual version of that class created using the "new" keywords

Use new when you want to interact with (or create) an object from a class.
Because new:
- Allocates memory for the object
- Initializes it using the class template
- Gives you a usable instance you can call methods on or store data in

class Car {
    String color;
    void drive() {
        System.out.println("Vroom!");
    }
}

Car myCar = new Car();  // ← This is the object
myCar.color = "Red";
myCar.drive();          // Output: Vroom!


Car → class (template)
myCar → object (real thing)

.getAbsolutePath()

Without getAbsolutePath()
File file = new File("photos/image.jpg");
System.out.println("Path: " + file);

= Path: photos/image.jpg


With getAbsolutePath()
File file = new File("photos/image.jpg");
System.out.println("Absolute Path: " + file.getAbsolutePath());

= Absolute Path: C:\Users\user\Desktop\xxx\Java\PhotoLinkManager\photos\image.jpg

Why declare "throws IOException"
- commonly used when a program read input from files or streams and these operations can encounter errors:
> File not found
> unable to read file contents
> Network connection issues (if dealing with networking streams)

eg

public static void main(String[] args){
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    try{
        String input = br.readLine();
        system.out.println("You typed: " + input);
    } catch(IOException e){
        System.out.println("An I/O error occurred: " + e.getMessage());
    }
}

or

public static void main(String[] args) throws IOException {
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String input = br.readLine();
    System.out.println("You typed: " + input);
}

readLine() is a method provided by Java's BufferedReader class. It reads a line of text from an input stream, such as from the console (keyboard) or a file.

InputStreamReader is a Java class used to bridge between byte streams and character streams. It converts bytes (typically from input sources like the keyboard, files, or network connections) into characters based on a specified or default character encoding (such as UTF-8).
- Byte Stream - Images, Video, Audio
- Chracter Stream - Documents, text files

BufferedReader - Efficiently reads characters or lines - (Type) Character Stream
